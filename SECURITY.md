# Security Policy

## Supported Versions

Security fixes are provided for the latest `main` branch state.

## Reporting a Vulnerability

Please do not open public issues for security vulnerabilities.

1. Send a private report to the maintainers via repository owner contact.
2. Include affected module, reproduction steps, impact, and proposed fix (if any).
3. If credentials, tokens, or personal data may be exposed, clearly mark severity.

## Response SLA

1. Initial response target: within 72 hours.
2. Triage and severity classification: within 7 days.
3. Fix timeline depends on severity:
   - Critical: as soon as possible, target 72 hours after confirmation.
   - High: target within 7 days after confirmation.
   - Medium/Low: included in next scheduled maintenance release.

## Disclosure

After fix release and verification, we will publish a security advisory with:

1. Affected versions and impact scope.
2. Mitigation and upgrade guidance.
3. Credit information (if reporter agrees).
